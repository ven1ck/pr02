# Документация проекта: Генератор QR-кодов

## 1. Описание проекта
Веб-приложение на базе Django для генерации двумерных штрихкодов (QR-кодов) из произвольных текстовых данных. Сервис поддерживает настройку визуальных параметров, интеграцию логотипа, систему аутентификации и сохранение истории генераций для зарегистрированных пользователей. Интерфейс построен на Bootstrap 5, полностью адаптирован под мобильные устройства и поддерживает переключение светлой/темной темы.

## 2. Системные требования
- Python 3.10 или выше
- Менеджер пакетов pip
- Виртуальное окружение (рекомендуется)
- Современный веб-браузер

## 3. Установка и запуск
1. Перейдите в корневую директорию проекта.
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/macOS:
   source .venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Примените миграции базы данных:
   ```bash
   python manage.py migrate
   ```
5. Создайте учетную запись администратора:
   ```bash
   python manage.py createsuperuser
   ```
6. Запустите сервер разработки:
   ```bash
   python manage.py runserver
   ```
7. Откройте в браузере: `http://127.0.0.1:8000/`

## 4. Конфигурация
Настройки проекта находятся в `project/settings.py`.
- `SECRET_KEY`: Обязателен к замене в продакшене.
- `DEBUG`: Установите `False` перед публикацией.
- `ALLOWED_HOSTS`: Укажите домены или IP-адреса сервера.
- База данных: по умолчанию используется SQLite (`db.sqlite3`). Для высокой нагрузки рекомендуется заменить на PostgreSQL.
- Медиафайлы сохраняются в директории `media/`, статика обслуживается через `static/`.

## 5. Функциональные возможности
### 5.1. Генерация QR-кода
- Поле ввода для текста, URL или контактных данных.
- Выбор размера изображения (200–600 пикселей).
- Уровень коррекции ошибок: Низкий, Средний, Квартиль, Высокий.
- Настройка цветов переднего и заднего плана через палитру.
- Опциональная загрузка логотипа для размещения в центре кода (автоматическое масштабирование до 20% от размера кода).
- Мгновенный предпросмотр результата на странице генерации.
- Кнопка скачивания сгенерированного файла в формате PNG.

### 5.2. Аутентификация и регистрация
- Форма входа с валидацией и адаптивным интерфейсом.
- Форма регистрации с полями: имя пользователя, email, пароль, подтверждение пароля.
- Автоматический вход после успешной регистрации.
- Возможность выхода из системы через навигационную панель.

### 5.3. История генераций
- Доступна только авторизованным пользователям через пункт меню `История`.
- Отображает таблицу с датой создания, содержимым кода, размером и уровнем коррекции.
- Предусмотрена возможность повторного скачивания каждого кода.
- Пустое состояние обрабатывается информативным плейсхолдером.

### 5.4. Интерфейс и адаптивность
- Сетка `col-12 col-lg-7` (настройки) и `col-12 col-lg-5` (результат) на десктопе.
- На мобильных устройствах блоки выстраиваются вертикально.
- Все поля ввода и кнопки соответствуют стандартам доступности и минимальному размеру области нажатия (48px).
- Поле размера открывает цифровую клавиатуру на мобильных устройствах (`inputmode="numeric"`).
- Переключатель темы в навигационной панели сохраняет выбор в `localStorage`.

## 6. Административная панель
Доступ по адресу: `http://127.0.0.1:8000/admin/`

### 6.1. Управление пользователями
- Стандартный интерфейс `UserAdmin` переопределен.
- Убраны разделы `Группы` и `Права пользователей` для упрощения интерфейса.
- При создании пользователей через админку отключена строгая валидация паролей (допускаются любые значения при наличии прав суперпользователя).

### 6.2. Управление QR-кодами
- Модель `QRCode` зарегистрирована в адинтерфейсе.
- Доступные действия: просмотр списка, фильтрация по пользователю/дате/коррекции, поиск по тексту, сортировка по дате создания, массовое удаление.
- Поле `image` недоступно для прямой загрузки в админке (генерируется автоматически через представление).
